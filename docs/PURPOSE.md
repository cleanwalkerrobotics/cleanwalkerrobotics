# docs — Purpose

Research, strategy, sales materials, and technical documentation.

**Owner:** all teams
**What belongs here:** Research papers, CEO strategy docs, sales pitch decks, design specs, regulatory compliance, market research, grant applications, technical references.
**What does NOT belong here:** Code documentation (keep near the code), operational runbooks (→ ops/), API docs (auto-generate from code).
**Sub-areas:** research/, ceo/, sales/, design/, technical/, regulatory/, market-research/.
